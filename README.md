## Local Development with DDEV

### Prerequisites

- [DDEV](https://ddev.readthedocs.io/) installed
- Docker running
- Composer installed

### Quick start

1. **Clone this repository:**
   ```bash
   git clone https://github.com/YOUR-USERNAME/demo-tugboat-drupal-cms.git
   cd demo-tugboat-drupal-cms
   ```

2. **Start DDEV:**
   ```bash
   ddev start
   ```

3. **Install dependencies:**
   ```bash
   ddev composer install
   ```

4. **Import configuration and install the site:**
   ```bash
   ddev drush site:install --existing-config -y
   ```

5. **Visit your site:**
   ```bash
   ddev launch
   ```

6. **Login as admin:**
   ```bash
   ddev drush uli
   ```

## Configuration management

This project uses Drupal's configuration management system with the config directory at `/config` (outside the web root for security).

**Export configuration after changes:**
```bash
ddev drush config:export -y
```

**Import configuration:**
```bash
ddev drush config:import -y
```

## Available content types

- **Goat profile** - Custom content type for Tubgoat crew members
- **Page** - Standard Drupal CMS page content type. The /tubgoat page was created like the Person Profile page that comes out-of-the-box with Drupal CMS. I duplicated the Person profiles view (with block display) and updated it to use Goat profiles. Then I created a Basic page and edited the layout, adding the Goat profile block (listing Goat profiles) to the appropriate section. This way every time a new Goat profile is created, it's listed automatically on this page.
